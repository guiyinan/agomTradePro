# VPS Deployment Bundle

This bundle is generated by `scripts/package-for-vps.ps1` and deployed on Linux VPS by:

- `scripts/deploy-on-vps.sh` (recommended)
- `scripts/deploy-on-vps.ps1` (PowerShell 7 on Linux)

## Quick Start (Linux)

1. Copy bundle tar.gz to VPS.
2. Extract and enter directory.
3. Run:

```sh
bash scripts/deploy-on-vps.sh --bundle ./agomsaaf-vps-bundle-<tag>.tar.gz
```

## Backup and Restore (VPS)

Create backup:

```sh
bash scripts/vps-backup.sh --target-dir /opt/agomsaaf/current --backup-dir /opt/agomsaaf/backups --keep-days 14
```

Restore latest backup:

```sh
bash scripts/vps-restore.sh --target-dir /opt/agomsaaf/current --backup-dir /opt/agomsaaf/backups
```

## What is included

- Docker image tars for app + postgres + redis + caddy
- Docker image tars for app + redis + caddy + rsshub
- `docker/docker-compose.vps.yml`
- `docker/Caddyfile.template`
- `.env` template
- Optional backups:
  - `backups/db.sqlite3`
  - `backups/dump.rdb`
- `deploy/manifest.json` with SHA-256 checksums

## Notes

- Scripts are ASCII-only.
- Domain is optional. If omitted, Caddy is configured for HTTP only (`:80`).
- App runtime uses `core.settings.production`.
- `ENABLE_RSSHUB=true` enables self-hosted RSSHub service in the stack.
- `ENABLE_CELERY=false` is recommended on 4G VPS, enable only when needed.
- Local build/download cache is stored in `.cache/pip-wheels` (gitignored).
