# 决策平台用户指南

> **版本**: 1.0
> **更新日期**: 2026-02-05
> **面向用户**: 投资者、交易员、基金经理

---

## 一、平台概述

决策平台由三个核心模块组成，帮助您在正确的宏观环境中做出明智的投资决策：

### 模块组成

| 模块 | 功能 | 使用场景 |
|------|------|----------|
| **Beta 闸门** | 资产可见性过滤 | 防止在错误环境下交易 |
| **Alpha 触发器** | 离散信号触发 | 捕捉高质量投资机会 |
| **决策配额** | 频率约束管理 | 避免过度交易 |

### 统一工作台

访问 **决策工作台** (`/decision/workspace/`) 可以一站式查看：
- 当前 Regime/Policy 状态
- 可操作的 Alpha 候选
- 待处理的决策请求
- 配额使用情况
- 系统告警信息

## 二、Beta 闸门配置

### 2.1 什么是 Beta 闸门？

Beta 闸门基于当前的宏观环境（Regime + Policy）自动过滤资产可见性。当环境不利时，某些资产类别将被隐藏，防止冲动交易。

### 2.2 配置步骤

1. **访问配置页面**
   - 导航到 `/beta-gate/config/`
   - 或在 Django Admin 中进入 "Beta Gate 配置"

2. **设置风险画像**
   ```
   风险画像描述您的投资风格：
   - CONSERVATIVE（保守）：仅允许股票、债券
   - BALANCED（平衡）：允许股票、债券、部分商品
   - AGGRESSIVE（激进）：允许所有资产类别
   ```

3. **配置 Regime 约束**
   - 选择当前 Regime（复苏/过热/通缩/滞胀）
   - 设置置信度阈值（0-1）
   - 选择允许的资产类别

4. **配置 Policy 约束**
   - 设置当前政策档位（P0-P3）
   - 设置最大风险敞口
   - 添加硬性排除（如：期货、期权）

5. **配置组合约束**
   - 最大持仓数量
   - 单个持仓最大权重
   - 集中度上限

### 2.3 版本管理

- 每次修改配置会自动创建新版本
- 使用 **版本对比** 功能查看差异
- 支持一键回滚到历史版本

### 2.4 资产测试

使用 **资产测试工具** (`/beta-gate/test/`)：
1. 输入资产代码（如：000001.SH）
2. 选择资产类别
3. 点击"测试"
4. 查看结果（通过/拦截原因）

## 三、Alpha 触发器管理

### 3.1 什么是 Alpha 触发器？

Alpha 触发器是预定义的投资信号规则，当条件满足时自动生成候选。每个触发器都包含：
- **触发条件**：什么情况下触发
- **证伪条件**：什么情况下失效
- **置信度**：信号可靠性（0-1）
- **投资论点**：为什么这个信号有效

### 3.2 创建触发器

1. **访问创建页面**
   - 导航到 `/alpha-triggers/create/`
   - 点击"创建触发器"

2. **选择触发器类型**
   ```
   THRESHOLD_CROSS    - 阈值穿越
   MOMENTUM_SIGNAL    - 动量信号
   REGIME_TRANSITION  - Regime 转换
   POLICY_SHIFT       - 政策变化
   MEAN_REVERSION     - 均值回归
   ```

3. **填写基本信息**
   - 资产代码（如：000001.SH）
   - 资产类别（如：a_股票）
   - 方向（LONG/SHORT/NEUTRAL）
   - 投资论点（描述逻辑）

4. **配置触发条件**
   - 选择指标（如：CN_PMI_MANUFACTURING）
   - 设置阈值和方向
   - 配置参数

5. **配置证伪条件**（重要！）
   - 使用**可视化构建器**
   - 选择条件类型（指标/时间/Regime/政策）
   - 设置阈值和操作符
   - 添加多个条件并配置 AND/OR 逻辑

6. **设置置信度和有效期**
   - 置信度：0-1（建议 ≥ 0.6）
   - 有效期：触发器存续时间（天）

7. **保存并测试**
   - 点击"创建"保存触发器
   - 系统会立即开始监控

### 3.3 证伪规则可视化

使用 **证伪规则构建器** (`/alpha-triggers/invalidation-builder/`)：

1. **添加条件**
   - 点击"添加条件"
   - 选择条件类型

2. **条件类型**
   ```
   指标穿越：指标值突破阈值
   时间衰减：超过一定时间自动失效
   Regime 不匹配：环境变化时失效
   政策变化：政策调整时失效
   止损触发：价格触及止损线
   目标达成：价格达到目标
   ```

3. **设置参数**
   - 指标代码（如：CN_PMI_MANUFACTURING）
   - 阈值（如：49.5）
   - 方向（大于/小于/等于）

4. **配置逻辑**
   - AND：所有条件都满足才证伪
   - OR：任一条件满足即证伪

5. **预览和保存**
   - 查看 JSON 格式预览
   - 点击"保存"应用规则

### 3.4 管理触发器

**查看所有触发器**
- 访问 `/alpha-triggers/list/`
- 按状态筛选（活跃/已触发/已失效）

**编辑触发器**
- 点击触发器操作栏的"编辑"
- 修改任何参数
- 保存更改

**查看详情**
- 点击触发器 ID 查看完整信息
- 查看相关候选列表
- 查看触发历史

### 3.5 性能追踪

访问 **性能追踪页面** (`/alpha-triggers/performance/`)：

**关键指标**
- **转化率**：候选转为执行的比例（≥ 50% 为优秀）
- **证伪率**：候选被证伪的比例（≤ 30% 为优秀）
- **平均持仓天数**：从候选到执行的时间
- **综合评分**：加权评分（≥ 70 分为优秀）

**优化建议**
- 高转化率 + 低证伪率 = 保持配置
- 低转化率 + 高证伪率 = 需要优化
- 高频率 = 考虑增加过滤条件

## 四、决策配额管理

### 4.1 什么是决策配额？

决策配额限制您的决策频率，防止过度交易。配额按周期重置：
- **每日配额**：每天重置
- **每周配额**：每周一重置
- **每月配额**：每月1号重置

### 4.2 查看配额

访问 **配额管理页面** (`/decision-rhythm/quota/`)：

**配额状态**
- 总配额：本期最大决策次数
- 已占用：已使用的决策次数
- 剩余额度：还可以提交的决策数
- 使用进度：可视化进度条

**冷却期**
- 显示当前处于冷却期的资产
- 冷却原因和剩余时间

**决策历史**
- 最近 30 天的决策记录
- 按状态筛选（待处理/已批准/已拒绝）

### 4.3 配置配额

访问 **配额配置页面** (`/decision-rhythm/config/`)：

**配置参数**
- 配额周期（日/周/月）
- 最大决策次数
- 最大执行次数
- 重置规则

**重置配额**
- 点击"重置配额"按钮
- 确认后立即清空已使用计数
- （仅管理员可用）

### 4.4 提交决策请求

**方式一：从候选提交**
1. 访问 `/alpha-triggers/list/`
2. 找到可操作的候选（ACTIONABLE 状态）
3. 点击"批准"或"执行"
4. 系统自动检查配额

**方式二：手动提交**
1. 访问 `/decision/workspace/`
2. 查看"决策待办"列表
3. 填写决策请求表单
4. 提交并等待批准

**优先级**
- HIGH（高）：紧急，优先处理
- MEDIUM（中）：正常优先级
- LOW（低）：可以延后处理

## 五、常见使用场景

### 场景 1：PMI 回升买入银行股

**步骤**：
1. 检查当前 Regime（复苏）
2. 配置 Beta Gate 允许股票
3. 创建 Alpha Trigger：
   - 类型：指标穿越
   - 指标：CN_PMI_MANUFACTURING
   - 条件：> 50 且连续 2 个月上升
   - 方向：LONG
   - 资产：银行 ETF 或龙头股
4. 证伪条件：PMI 跌破 50
5. 设置置信度 0.7

### 场景 2：CPI 过高做空债券

**步骤**：
1. 确认当前 Regime（过热）
2. 创建 Alpha Trigger：
   - 类型：指标穿越
   - 指标：CN_CPI_YOY
   - 条件：> 3%
   - 方向：SHORT
   - 资产：国债期货
3. 证伪条件：CPI 跌破 2.5%
4. 设置置信度 0.75

### 场景 3：政策收紧降低仓位

**步骤**：
1. 观察 Policy 档位（P2：收紧）
2. 配置 Beta Gate：
   - 降低风险敞口上限
   - 硬性排除高贝塔资产
3. 创建 Alpha Trigger：
   - 类型：Policy 变化
   - 条件：P0 → P2
   - 操作：减仓或清仓
4. 设置紧急触发

## 六、最佳实践

### 6.1 Beta Gate 配置

✅ **推荐做法**：
- 根据实际风险承受能力选择画像
- 定期回顾和调整配置
- 使用版本对比追踪变更

❌ **避免**：
- 过于激进的硬性排除
- 忽视 Regime 变化
- 频繁调整配置

### 6.2 Alpha Trigger 设计

✅ **推荐做法**：
- 设置合理的证伪条件
- 置信度 ≥ 0.6
- 投资论点清晰明确
- 定期检查性能指标

❌ **避免**：
- 过度拟合历史数据
- 证伪条件过于宽松
- 置信度设置过高
- 忽视时间衰减

### 6.3 决策频率控制

✅ **推荐做法**：
- 使用每周配额（默认 10 次）
- 高优先级留给好机会
- 定期回顾决策质量

❌ **避免**：
- 每日配额设置过低
- 无视冷却期
- 频繁手动重置配额

## 七、故障排查

### 问题 1：触发器没有生成候选

**可能原因**：
1. 触发条件未满足
2. 触发器已过期
3. 触发器被暂停

**解决方案**：
- 检查触发器状态
- 查看触发条件参数
- 延长有效期

### 问题 2：资产被 Beta Gate 拦截

**可能原因**：
1. 资产类别不在允许列表
2. 当前 Regime 不匹配
3. 组合约束限制

**解决方案**：
- 检查当前 Regime/Policy
- 调整 Gate 配置
- 等待环境改善

### 问题 3：配额不足

**可能原因**：
1. 本期决策次数过多
2. 配额设置过低

**解决方案**：
- 等待下周重置
- 提高配额上限
- 优先处理高价值机会

### 告警解读

| 告警类型 | 含义 | 操作 |
|----------|------|------|
| 🚨 配额已耗尽 | 本期配额用完 | 等待重置或提高限额 |
| ⚠️ 配额即将耗尽 | 剩余 < 20% | 谨整交易频率 |
| ⏰ 候选即将过期 | 2天内过期 | 及时处理或更新 |
| ⚡ 可行动候选 | 有投资机会 | 查看详情并决策 |
| 🔥 高优先级请求 | 紧急待处理 | 优先处理 |
