# AgomTradePro 用户旅程 UX 检查清单（2026-02-18）

## 1. 使用说明

- 目标：按真实使用路径检查体验，而不是按页面逐个检查。
- 建议频率：每次版本发布前至少走查 1 次。
- 结果记录：每项标记 `通过 / 部分通过 / 不通过`，并附问题链接。

---

## 2. 旅程 A：新用户入门

### A1 注册与登录

- [ ] 注册页字段说明清楚，错误提示可理解（不只是“失败”）。
- [ ] 协议与风险提示可读、可滚动、可确认。
- [ ] 登录后跳转路径符合预期（默认到 Dashboard）。
- [ ] 未登录访问受限页面时，能正确跳转到登录页。

### A2 首次理解系统

- [ ] Dashboard 首屏能回答 3 个问题：当前环境、当前仓位、下一步建议。
- [ ] 左侧导航分组命名与业务词汇一致（不产生歧义）。
- [ ] 关键卡片（Regime/Policy/配额）颜色与状态含义一致。

---

## 3. 旅程 B：研究与选标的

### B1 宏观研判

- [ ] Regime 页支持切换时点、数据源，状态反馈及时。
- [ ] Policy 页能快速看到“当前档位 + 最近事件 +建议”。
- [ ] Macro/Filter 页的筛选、刷新、加载状态明确。

### B2 资产筛选

- [ ] 资产筛选页（Asset/Fund/Equity）筛选条件分组清晰。
- [ ] 筛选后结果区更新及时，空结果有明确提示与建议。
- [ ] 从列表进入详情、返回列表时，筛选条件不丢失（或可快速恢复）。

---

## 4. 旅程 C：决策与执行

### C1 策略与信号

- [ ] 策略创建流程顺序合理（基础信息->风控->规则->脚本/AI）。
- [ ] 信号页面中“创建/审批/拒绝/证伪”动作入口清晰，避免误触。
- [ ] 关键动作有确认与结果反馈（成功、失败、原因）。

### C2 决策平面

- [ ] Decision Workspace 中 Beta/Alpha/Quota 三块状态一屏可读。
- [ ] 配额不足、候选即将过期等告警显著且可操作。
- [ ] Beta Gate 测试结果解释清晰（为什么通过/不通过）。

---

## 5. 旅程 D：交易与持仓管理

### D1 模拟交易

- [ ] 模拟盘入口页能快速到“我的账户/持仓/交易”。
- [ ] 账户详情页核心指标（总资产、收益、仓位）视觉层级明显。
- [ ] 持仓和交易表格支持基本筛选/排序/刷新，且操作有反馈。

### D2 账户管理

- [ ] Profile 与 Settings 的职责边界清晰（信息查看 vs 参数配置）。
- [ ] 资金流水录入/修改后的状态同步及时。

---

## 6. 旅程 E：复盘与运营

### E1 回测与审计

- [ ] 回测列表到详情路径清晰，结果页关键指标可快速定位。
- [ ] 审计页（归因/阈值/指标表现）图表、表格、说明术语一致。
- [ ] “应用回测结果”属于高风险动作，确认与回滚提示明确。

### E2 运营后台

- [ ] Ops Center 聚合入口完整且无失效链接。
- [ ] 文档管理、日志、AI Provider、Prompt 管理页面操作闭环完整。
- [ ] 管理类页面分页/筛选/搜索行为一致。

---

## 7. 全局体验基线（跨旅程）

- [ ] 导航一致性：顶部导航、侧边导航、面包屑逻辑一致。
- [ ] 视觉一致性：按钮、表单、表格、状态标签样式统一。
- [ ] 反馈一致性：加载中、成功、失败、空状态都可感知。
- [ ] 权限一致性：访客/用户/staff/superuser 的行为边界清晰。
- [ ] 文案一致性：同一概念只使用一套术语（如“配额”“候选”“档位”）。
- [ ] 健壮性：所有入口无 404，无明显 JS 报错，无阻断流程异常。

---

## 8. 本轮优先关注（结合当前代码盘点）

- [ ] Dashboard 侧栏链接有效性回归（宏观数据、股票分析）。
- [ ] 决策平面核心动作的确认与结果提示完整性。
- [ ] 模块间 UI 风格一致性（内联样式页面优先整改）。
