## 1. 概述

本文档定义了 AgomTradePro UI/UX 改进项目的用户验收测试（UAT）执行计划，包括测试范围、验收标准、执行流程和报告要求。

### 1.1 测试目标

1. 验证用户旅程 A~E 主链路可正常走通
2. 确认核心页面视觉一致性
3. 验证主导航无 404 链接
4. 验证 API 路由命名符合规范
5. 记录并跟踪 P0/P1 缺陷

### 1.2 验收标准

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 关键旅程通过率 | >= 90% | 自动化测试 + 手工验证 |
| 主导航 404 数量 | 0 | 链接检查脚本 |
| P0/P1 缺陷数量 | <= 2 | 缺陷跟踪表 |
| API 命名规范覆盖率 | 100% | API 合规性检查 |

## 4. 测试执行流程

### 4.1 准备阶段

1. **环境准备**
   ```bash
   # 启动开发服务器
   python manage.py runserver

   # 准备测试数据
   python manage.py loaddata test_data.json
   ```

2. **测试工具验证**
   ```bash
   # 验证 Playwright 安装
   pytest --version
   playwright install chromium

   # 验证测试配置
   python tests/uat/run_uat.py --help
   ```

### 4.2 执行阶段

#### 自动化测试执行

```bash
# 执行全部 UAT 测试
python tests/uat/run_uat.py

# 执行特定旅程
python tests/uat/run_uat.py --journeys=A,B

# 仅生成报告
python tests/uat/run_uat.py --generate-report
```

#### 手工测试检查项

1. **视觉一致性** - 截图对比
2. **交互反馈** - 加载状态、动画
3. **文案一致性** - 术语使用
4. **响应式布局** - 不同分辨率

### 4.3 报告阶段

1. 生成自动化测试报告
2. 整理手工测试结果
3. 汇总缺陷清单
4. 生成验收报告

## 5. 缺陷管理

### 5.1 缺陷分级

| 级别 | 定义 | 示例 | 发布阈值 |
|------|------|------|----------|
| P0 | 阻断性问题 | 主导航 404、登录失败 | 0 个 |
| P1 | 严重问题 | 关键功能无法使用 | <=2 个 |
| P2 | 一般问题 | UI 不一致、文案错误 | 可接受 |
| P3 | 优化建议 | 体验改进 | 可延后 |

### 5.2 缺陷报告模板

```markdown
## 缺陷 #{ID}

**标题**: {简短描述}

**级别**: P0/P1/P2/P3

**模块**: {模块名}

**环境**: {浏览器、分辨率}

**复现步骤**:
1. {步骤1}
2. {步骤2}
3. {步骤3}

**预期结果**: {预期行为}

**实际结果**: {实际行为}

**截图**: {截图路径}

**附加信息**: {日志、错误信息}
```

## 6. 验收决策

### 6.1 通过条件

同时满足以下条件方可通过验收：

1. 关键旅程通过率 >= 90%
2. 主导航 404 数量 = 0
3. P0 缺陷 = 0
4. P1 缺陷 <= 2
5. API 命名规范覆盖率 = 100%

### 6.2 不通过处理

如验收不通过：

1. 记录所有阻断性问题
2. 按优先级分配修复任务
3. 修复后进行回归测试
4. 重新执行验收流程
